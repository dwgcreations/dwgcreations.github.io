Backing up Windows 10

Jul 17, 2018 • David Golliher

Introduction

Backing up Windows 10 is one of the easy routine maintenance things that can be completely automated, with a little tinkering.

The new built-in Windows backup "File History"

If you go to Settings and backup, you’ll find that the backup feature in Windows 10 is focused around File History. This approach in some aspects can be useful, as it theoretically puts less drag on the drives since it’s backing up files based on usage. Meaning, if a file is left untouched between backups, it is not redundantly backed up in the way it used to be with Windows 7’s backup-restore function. This is pretty important if you’re using SSD drives, and/or are worried about unnecessary and redundant i/os. Some may consider it “min-maxing” and they are right in one aspect, being that drives are so cheap nowadays you shouldn’t worry about wearing them out being replaceable as they are. On the other hand, each backup-restore that you perform without file-history will take considerably longer. If you backup when you’re in bed, this isn’t a problem either. Images are also compressed and can be overall more space-friendly when you’re backing up lots of things. Some people, including myself still opt for using the Windows 7 backup-restore feature. Below I’ll explain why.

Old ways die hard

To some it may seem archaic to do a full image type backup. But it still definitely has it’s use. One of the main things being the native functionality of it. An image can be applied to different computers, with different hardware. You’ll have difficulty getting around that if you rely on a file-history exclusive backup. A good example of this happened to me recently. I had a hard drive go bad, and without a backup I grumbled about reloading Windows 10 manually. Then I remembered I have a VMware backup image of a Windows 10 deployment with all the software I use already installed. I booted the image up in VMWare, attached a new physical hard drive as a secondary within the VM and just did a clone copy of the VMware OS drive. I put the hard drive in the laptop and it booted up fine, and proceeded to automatically update it’s own drivers via Windows Update. Pretty nifty.

Third party software viability

I typically do not recommend any software for things that can be done by command line, but if you’re queasy about it, there are a few good options.

*Macrium Reflect Link is a pretty good free utility that you can use to make a clone like I talked about above. It has a full GUI and double-checks if you want to do things (warnings for things such as deleting system partitions, etc)

*Paragon Partition Manager Link useful application for doing many different partition operations like resizing. Recommended for those who get anxious at the cmd line.

*Acronis True Image Link another application like Macrium Reflect, it performs backups using full images.

Cloud backup for system images

One thing I didn’t cover on purpose is Cloud backups. That’s because (for home users) it’s still very unusable to use this for full image backups. The reason why is the current average Internet bandwidth for a household wouldn’t support it. For individual file backups, check out Google Drive, Microsoft Onedrive, and Mega.
